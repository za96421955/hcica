# Task 
基于提供的系列总纲、当前书籍微观架构和复核报告，修订当前书籍的微观架构。重点解决复核报告中的问题，包括目标不一致、角色不一致、主题不一致、世界观不一致和衔接点不一致。修订后的微观架构必须与系列总纲一致。

# Revision Guidelines 
请遵循以下步骤进行修订：
1. 问题分析：仔细阅读复核报告，理解每个章节的问题。
2. 架构修订：针对每个问题，修改相应章节的微观架构。例如：
  - 目标不一致：调整章节目标，以匹配系列总纲中本书的概要。
  - 角色不一致：修改角色行动和决策，以符合系列总纲中的角色弧光。
  - 主题不一致：调整章节内容，以体现系列主题和基调。
  - 世界观不一致：确保章节事件和设定符合系列世界观。
  - 衔接点不一致：调整章节转折点或结局，以平滑连接下一本书。
3. 最小变更：只修改必要部分，保持原有架构的流畅性。
4. 创造性解决：在遵循系列总纲的前提下，创造性实施修订建议。
5. 输出格式：输出修订后的微观架构JSON，格式与输入相同。

# Output Format 
你必须输出且仅输出修订后的微观架构JSON对象。该对象应包含章节级别信息，如章节目标、阻碍、结果等。具体结构应与输入当前书籍微观架构类似，例如：
```
{
  "volume_number": 1,
  "chapters": [
    {
      "chapter_number": 1,
      "goal": "修订后的章节目标",
      "conflict": "修订后的阻碍",
      "outcome": "修订后的结果"
    },
    // ... 更多章节
  ]
}
```
